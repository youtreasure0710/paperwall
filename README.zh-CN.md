# PaperWall

一个本地离线的论文管理与阅读小工具。  
也是我第一次正儿八经做一个“文献管理器”。

说实话，一开始只是想给自己做个顺手一点的论文工作台，结果越做越上头，慢慢从“能导入 PDF 就行”变成了现在这个样子。

如果你也刚好有囤论文、看论文、做摘录、记笔记的需求，欢迎试试看。  
如果你发现哪里很蠢、很怪、很不合理，也欢迎直接提 issue —— 毕竟这是我第一次做这类工具，很多地方大概率还能再救一下。

总之：

- 功能在努力变多
- bug 也可能很努力地一起变多
- 欢迎提建议
- 欢迎抓 bug
- 欢迎围观一个新手是怎么一边踩坑一边把项目做出来的

---

## PaperWall 现在能干什么？

目前它主要是一个 **本地离线论文管理 + 阅读 + 笔记工作台**，大概支持这些事情：

- 导入本地 PDF 论文，并托管到应用目录（不改你的原文件）
- 用卡片墙方式浏览论文
- 查看右侧详情信息
- 全文阅读
- 划线高亮
- 做结构化笔记 / 摘录
- 本地补全一些元数据（比如 DOI / arXiv）
- 导出引用

简单说，就是想把“论文堆在文件夹里懒得整理”的状态，尽量变成一个还算顺手的阅读工作台。

---

## 当前核心功能

- PDF 导入、托管复制、缩略图生成
- 搜索 / 筛选 / 排序
- 智能书架
- 分类管理
- 收藏、阅读状态管理
- 结构化笔记与摘录
- 全文阅读与文本高亮
- 引用导出（EndNote RIS / GB-T）
- 打开 PDF / 打开文件夹
- 重复检测、删除论文

## 本地开发

### 环境要求

- Node.js 20+
- npm 10+
- Rust stable
- macOS（目前优先在 macOS 上开发和测试）

### 安装依赖

```bash
npm install
````

### 启动开发环境

```bash
npm run tauri:dev
```

### 构建检查

```bash
npm run build
cd src-tauri && cargo check
```

---

## 打包（macOS）

### 生成 DMG

```bash
npm run tauri:build:mac:dmg
```

这个命令会先构建 `.app`，再生成 `.dmg`。

### 产物路径

* App: `src-tauri/target/release/bundle/macos/PaperWall.app`
* DMG: `src-tauri/target/release/bundle/dmg/PaperWall_<version>_<arch>.dmg`

## macOS 提示（第一次打开）

因为当前还是测试阶段，应用暂时没有签名 / notarize，
所以第一次打开时，macOS 可能会弹出“未识别开发者”之类的提示。

可以这样放行：

1. 在 Finder 里右键 `PaperWall.app` -> `打开`
2. 或者去 **系统设置 -> 隐私与安全性**，点击“仍要打开”

这不是它坏了，主要是我还没把“苹果开发者全家桶”那套完全补齐。
